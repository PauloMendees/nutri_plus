# 03.1 - Patient Domain Model

## Goal

Define all patient-related entities, fields, relationships, and business rules used throughout the platform.

This document is the source of truth for:

- Patient management
- Meal plan generation
- AI recommendations
- Progress tracking
- Outside-home feature
- Future analytics and reporting

---

# Overview

Patient data is divided into two categories:

## Static Information

Information that rarely changes:

- Date of birth
- Gender
- Activity level
- Allergies
- Medical conditions

Stored in:

```txt
PatientProfile
```

## Dynamic Information

Information that changes over time:

- Weight
- Body fat percentage
- Muscle mass
- Circumferences
- Bioimpedance measurements

Stored in:

```txt
BodyAssessment
```

This allows the platform to maintain a complete patient history.

# PatientProfile

Represents the patient's permanent profile.

## Prisma Model

```prisma
model PatientProfile {
  id                    String @id @default(uuid())

  userId                String @unique
  user                  User
                         @relation(fields: [userId], references: [id])

  nutritionistId        String?
  nutritionist          NutritionistProfile?
                         @relation(fields: [nutritionistId], references: [id])

  birthDate             DateTime?

  gender                Gender?

  height                Float?

  targetWeight          Float?

  objective             PatientObjective?

  activityLevel         ActivityLevel?

  restrictions          String?

  allergies             String?

  medicalConditions     String?

  notes                 String?

  createdAt             DateTime @default(now())
  updatedAt             DateTime @updatedAt

  assessments           BodyAssessment[]
}
```

# Field Definitions

## birthDate

Patient date of birth.

Used by:

- AI recommendations
- Calorie calculations
- Reports

## gender

Patient biological or declared gender.

Used by:

- TDEE calculations
- Reports
- AI context

## height

Height in centimeters.

Example:

```txt
178
```

## targetWeight

Desired body weight in kilograms.

Example:

```txt
75.0
```

## objective

Patient goal.

Examples:

- Weight loss
- Muscle gain
- Maintenance
- Recomposition

## activityLevel

Represents daily activity level.

Used by:

- Calorie calculations
- AI recommendations

## restrictions

General dietary restrictions.

Examples:

```txt
No pork
Vegetarian
Vegan
```

## allergies

Food allergies.

Examples:

```txt
Peanut
Shellfish
Lactose
```

## medicalConditions

Relevant health conditions.

Examples:

```txt
Type 2 Diabetes
Hypertension
Hypothyroidism
```

## notes

Private nutritionist notes.

Not exposed to AI by default.

# BodyAssessment

Stores all progress-related information and bioimpedance measurements.

Each record represents a specific assessment date.

## Prisma Model

```prisma
model BodyAssessment {
  id                    String @id @default(uuid())

  patientId             String
  patient               PatientProfile
                         @relation(fields: [patientId], references: [id])

  assessmentDate        DateTime @default(now())

  weight                Float?

  bodyFatPercentage     Float?

  muscleMass            Float?

  leanMass              Float?

  visceralFat           Float?

  basalMetabolicRate    Float?

  bodyWaterPercentage   Float?

  boneMass              Float?

  metabolicAge          Int?

  waistCircumference    Float?

  hipCircumference      Float?

  chestCircumference    Float?

  armCircumference      Float?

  thighCircumference    Float?

  notes                 String?

  createdAt             DateTime @default(now())
}
```

# Business Rules

## Rule 1

A patient may have many BodyAssessment records.

## Rule 2

Assessments are immutable historical records.

Avoid updating old assessments.

Create a new assessment instead.

## Rule 3

The latest assessment is considered the current body composition snapshot.

## Rule 4

Only linked nutritionists can create assessments.

## Rule 5

Patients may view their assessments but cannot edit them.
