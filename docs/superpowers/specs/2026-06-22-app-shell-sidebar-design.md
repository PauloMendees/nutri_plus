# Logged-in App Shell + Sidebar Navigation — Design

**Date:** 2026-06-22
**Status:** Approved (pending implementation plan)
**Scope:** Replace the current top-header `(app)` layout with a sidebar-based app shell and three clickable module routes (Pacientes, Funcionários, Agenda) as stub pages.
**Builds on:** the web foundation/auth slice (`docs/superpowers/specs/2026-06-21-web-foundation-and-auth-design.md`). Same stack: Next.js App Router + shadcn + Tailwind + the iNutri brand. Branch `feat/web-foundation-auth`.

---

## 1. Goal

A nutritionist who logs in lands inside an application shell with a persistent left **sidebar** for navigation. The sidebar exposes the three main modules — **Pacientes**, **Funcionários**, **Agenda** — each clickable to its own route. This slice delivers the shell + navigation + stub module pages; the modules' real features are out of scope (separate specs later).

Done when: after login the user is taken to a module, sees the green branded sidebar with the three modules + the active one highlighted + their name and a sign-out control, can navigate between the (stub) modules, and on mobile opens/closes the sidebar via a top-right hamburger.

## 2. Locked decisions

| Decision | Choice |
|---|---|
| Sidebar implementation | **shadcn `Sidebar`** component (add via CLI) — accessible, collapsible, mobile sheet built-in |
| Module pages | **Stubs/placeholders** (title + empty state); real features deferred |
| Default landing (`/`) | **Redirect to the first module** (`/patients`) |
| Sidebar style | **Branded green** (deep-green `#0a5c45`, white reverse logo, teal active accent) |
| Mobile nav | **Hamburger at top-right** opens the sidebar as an overlay sheet (market standard) |
| Route slugs | English (`/patients`, `/employees`, `/agenda`) to match the API; labels pt-BR |
| Testing | Vitest + Testing Library, mirroring existing web tests |

## 4. Navigation config

`nav-items.ts` exports the ordered list, consumed by the sidebar (and the only place to edit to add modules later):

```
Pacientes    → /patients    → Users   (lucide)
Funcionários → /employees   → Briefcase
Agenda       → /agenda      → Calendar
```

Active state: `usePathname()` — an item is active when the pathname equals or starts with its `href`.

## 5. Behavior

- **Desktop:** the green sidebar is always visible and fixed (no collapse toggle). The content area (`SidebarInset`) shows the active module page; each stub page renders its own title header + empty state.
- **Mobile (<768px):** the sidebar is hidden; a slim top bar (`md:hidden`) shows the colored logo (left) + a **hamburger `SidebarTrigger` (right)**. Tapping it opens the sidebar as an overlay sheet (with scrim); tapping a nav item navigates and closes the sheet.
- **Sign out:** the footer control calls `supabase.auth.signOut()` → `router.push('/login')` (the existing sign-out behavior).
- **`/`:** redirects to `/patients`.
- Route protection is unchanged: every `(app)` route is non-public, so the middleware already requires a session (sessionless → `/login`).

## 6. Theming

The shadcn Sidebar reads dedicated CSS tokens. In `globals.css` (`:root` + the `@theme inline` map) set them to the brand:

- `--sidebar` = `#0a5c45` (deep green); `--sidebar-foreground` = `#ffffff`.
- `--sidebar-accent` = a lighter green (e.g. `#0e7a5c`) for hover/active backgrounds; `--sidebar-accent-foreground` = `#ffffff`.
- `--sidebar-primary` = teal `#14bfa6`; `--sidebar-primary-foreground` = `#04241b`.
- `--sidebar-border` = a translucent-reading green line; `--sidebar-ring` = `#14bfa6`.

The active item additionally gets a teal left accent (e.g. `inset 2px 0 0 var(--sidebar-ring)` via a className on the active `SidebarMenuButton`). The header logo uses `<Logo variant="full" tone="reverse" />` (white).

## 7. Testing (Vitest + Testing Library)

- **`app-sidebar`** — renders the three nav links with correct `href`s (`/patients`, `/employees`, `/agenda`) and labels; renders the iNutri logo and the user's name/role; marks the item matching a mocked `usePathname()` as active (`aria-current`/active styling); the footer sign-out triggers `signOut` then `router.push('/login')`. (Mock `next/navigation` `usePathname`/`useRouter` and `@/lib/supabase/client`; render inside `SidebarProvider`.)
- **`(app)/page`** — calls `redirect('/patients')` (mock `next/navigation` `redirect`).
- **`page-placeholder`** — renders the given title + description + empty-state copy.
- Stub module pages are thin wrappers over `PagePlaceholder`; the placeholder test covers their content. The server `layout.tsx` (session + `getMe`) is integration-level and verified via `tsc`/`build`, not a unit test.

## 8. Out of scope (YAGNI)

Real module features (patient list/CRUD, employee management, calendar/appointments UI) — each its own spec. Role-gating the **Funcionários** item (all three visible for now). Desktop collapse-to-icons. Breadcrumbs / global search / notifications in the shell.
